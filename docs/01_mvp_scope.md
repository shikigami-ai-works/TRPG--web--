# MVPスコープ

## MVPの目的

最初のバージョンでは、ゲームの核が面白いかを検証する。

- 探索者を作れる
- 短いシナリオを遊べる
- AI NPCと会話できる
- ダイス判定がある
- クリアまたはロストがある
- NPC好感度が変化する
- 条件を満たすとNPC連絡先を入手できる
- シナリオ終了後に結果が記録される
- 簡易エンディングツリーが更新される

## MVPに入れるもの

### プレイヤーキャラクター作成

- 名前
- 外見メモ
- 性格メモ
- 能力値
- ステータス: active / lost / memorial
- 所持アイテム
- シナリオ履歴

### 短いシナリオ1本

構成:

- 導入
- 探索シーン
- NPC会話シーン
- 判定イベント
- 危険イベント
- 分岐
- エンディング

### AIロール補助NPC

MVPではNPCは2〜3人。
うち1人は好感度と連絡先システムを持つ。

### ダイス判定

最初は「能力値 + ダイス >= 目標値」で判定する。
死に関わる判定は補正を入れて甘くする。

### NPC連絡先

条件:

- 好感度が一定以上
- NPCがロストしていない
- シナリオをクリアした

## MVPでは後回しにするもの

- 本番課金
- 複数シナリオ
- 高度なAI記憶
- 複雑な戦闘
- NPC同伴の完全実装
- 外見変更アイテム
- 有料シナリオ販売
- 高度なエンディングツリーUI
