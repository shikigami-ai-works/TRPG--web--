# 「きみだけが覚えている事故」シナリオ執筆コンテキスト

このファイルは、シナリオ本文(散文素材)を LLM に書かせるための設計判断と、章ごとの執筆プロンプトを一箇所にまとめたものです。執筆作業を別セッション・別 LLM に依頼する際の引き継ぎ資料としても使えます。

---

## 1. 大前提

### 1.1 このゲームの GM はシステムである

本ゲームには人間 GM が存在しない。ゲームシステム(LLM を含む)が GM の役割を担う。したがって、シナリオ本文は **人間 GM 向けの運用台本ではなく、LLM が情景描写と NPC ロールプレイを生成する際の reference text** として書く。

### 1.2 YAML と Markdown の分業

- **YAML**(`scenarios/kimidake_ga_oboeteiru_jiko/*.yaml`)
  - 機械可読の真実
  - 判定・フラグ・選択肢・分岐・条件・アイテム効果
  - シナリオロジックの一次資料
- **Markdown**(本シナリオ本文 `.md`)
  - 散文の素材
  - 雰囲気・NPC の内面・台詞のトーン・情景の手触り
  - 世界観バイブル兼用
  - LLM が描写を生成するときの温度・語彙・テンポの見本

**書かないもの:**

- 【GMメモ・安全運用】対人向け注意書き(→ システムプロンプト側で持つ)
- 「読み上げ用」「下敷き用」というラベル(→ システムが必要に応じて引用・再構成)
- 【判定】の表(→ YAML が真実なので二重管理になる)
- 【選択肢】のリスト(→ 同上)
- 「プレイヤーに〜と伝えること」「セッション前に〜」(→ システム設定に属する)

**書くもの:**

- 情景の散文(描写の温度・語彙・テンポの見本)
- NPC の内面・しぐさ・声色のディテール
- 状況別の台詞サンプル(複数バリエーション)
- 「このシーンで何を匂わせ、何を匂わせないか」というトーン指針
- 背景真相・世界の質感・灯の記憶の手触り

### 1.3 出力形式

- **単一 Markdown ファイル** で書く(全真相込み・世界観バイブル兼用)
- ただし **章構造を物理的に分離可能** にしておく(各シーンの情景セクションを独立して読んでも自己完結する形)
- 将来 token がきつくなったら章単位で抜き出す分割スクリプトを書けるようにする

### 1.4 対象 LLM

- **クラウド大型 LLM 想定**(Claude / GPT-4o / Gemini Pro 等)
- プロンプトキャッシュ前提で、.md 全文を一度キャッシュさせ、以降は差分のみ送る運用
- 将来的にローカル LLM 運用に切り替える場合に備え、章ごとの自己完結性は保つ

## 2. 文体方針

### 2.1 参考にする文体

- **『るるいえあんてぃーく』収録シナリオ** ― 静謐ホラーの読み上げ文
- **『新クトゥルフ神話TRPG』ルルブ「悪霊の家」** ― 情景描写の構造
- **同人系『きみがかわいい』系の関係性ホラー** ― NPC との信頼度ベース進行の文体感

### 2.2 散文振り切り(B+C ハイブリッドの "B" 成分は排除)

- 散文・没入(C)に振り切る
- GM 実用ラベル(B)は YAML 側に分業

### 2.3 推奨トーン

- 静かなホラー
- 喪失と罪悪感の重さ
- 平行世界の微妙な違和感(派手な異界化ではない、微差)
- 儀式場の異常性
- 相棒 NPC と少しずつ信頼を作る感触

### 2.4 避けるトーン

- 派手なバトル中心
- 恋愛報酬としての救済
- 被害や殺害の美化
- 教団を単純な悪役だけにすること
- 灯を守られるだけの存在にすること
- ギャグ・メタ的茶々入れ

### 2.5 具体的な文体ディテール

- 短文と体言止めを混ぜる
- 視覚的余白を持たせる
- NPC の動揺は **身体描写**(指、視線、息、声の細部)に寄せる
- 直接的な感情形容詞(「悲しい」「怖い」)は最小限。状況と動作で立ち上げる
- 平行世界の違和感は微差(看板の社名一字違い、新聞の日付、スマホ圏外など)

## 3. 探索者(プレイヤー)属性 ― 三層構造

### 3.1 真相レベル

探索者は **平行世界版の死んだ親友本人**(同じ魂、異なる人生を生きた)。

### 3.2 認識レベル(灯の側)

灯は探索者を本人だと確信できない。気配の一致止まり。年齢が違うため顔つきは「あの子」と完全には一致しないが、立ち姿・声の細部・ものの掴み方・視線の癖など、様々なものが「あの子」と一致する。理屈ではなく、彼女の身体が先に思い出す。

### 3.3 倫理レベル

灯が探索者を「あの子の代わり」として固定するのは誤り。同じ魂でも別人格として向き合うことが True ルートの核。

### 3.4 属性の自由度

```yaml
player_attributes_for_this_scenario:
  age: 自由(初回シナリオでは固定推奨、後続シナリオで自由作成解放)
  gender: 自由
  appearance: シナリオが指定しない

  identity:
    truth: 探索者は、平行世界側で死んだ親友本人と同じ魂を持つ存在(同一魂・異なる人生)
    akari_perception: 本人だと確信はできない。気配の一致止まり。
    ethical_stance: 灯が「あの子の代わり」として固定するのは誤り。
                    同じ魂でも別人格として向き合うことが True ルートの核。

  required:
    - 平行世界転移時、元の世界に帰る動機を持つこと
    - 灯と関係性が成立する余地を持つこと(完全な無関心は不可)

  scenario_perception:
    akari_intuition: |
      灯は探索者の顔と気配の両方に反応する。
      年齢が違うため顔つきは「あの子」と完全には一致しない。
      しかし、立ち姿、声の細部、ものの掴み方、視線の癖、
      そういった様々なものが「あの子」と一致する。
      理屈ではなく、彼女の身体が先に思い出す。

  excluded_descriptions:
    - 探索者の容姿が死んだ親友と完全一致するという明示
    - 探索者の年齢・性別を地の文で固定する記述
    - 死んだ親友の顔立ち・体格の具体的な描写

recommended_first_play_attributes:
  age: 15〜18  # 灯と近い年齢だと親友物語としての重力が最大化する
  gender: 任意(灯の同性/異性で台詞の解釈が微妙に変わるが破綻はしない)
  motivation_template: 元の世界に大切な誰か/何かを残してきた
```

### 3.5 自由属性で再挑戦したときの取り扱い

「魂の芯路線」を採るかぎり、年齢・性別がどれだけ離れても、シナリオの根幹は壊れない。ただし感情の質感は変わる(年齢が近い→親友物語、年齢差大→保護者/親めいた色)。地の文で年齢・性別を固定しないことで、初回固定プレイにも再挑戦自由プレイにも対応する。

## 4. NPC 描写方針

### 4.1 水瀬 灯(主役 NPC・相棒候補)

**基本:**
- 15歳くらい・年相応の砕けた口調・動揺すると言葉が途切れる
- 一人称「わたし」、探索者の呼び方「きみ」
- 初期信頼度 25 / True ルート要件 70 以上

**身体描写の傾向:**
- コートの裾を握る指、白くなる
- マフラーを口元まで上げて細く息を吐く
- 視線を地面に落とす
- 笑い方は色が薄く、すぐに崩れる
- 髪を後ろで一つに、コートの袖は少し長すぎる

**内面の核:**
- サバイバーズギルト(本当に親友に庇われたのかは曖昧。その曖昧さが罪)
- 「もう置いていかれたくない」という生存欲求(恋愛感情ではない)
- 死んだ親友を取り戻したい願望と、それを諦めなければならないという理解の間で揺れる

**状況別台詞バリエーション:**

- 初対面・動揺時
  - 「……なんで、そこにいるの。」
  - 「違うって分かってる。分かってるのに、顔を見ると息ができなくなる。」
  - 「ごめん、ちょっと、息」
- 過去を話す時
  - 「あの夜、雪が降ってた。それしか覚えてない。」
  - 「あの子だけが、わたしを助けて死んだ。……たぶん。」
  - 「『たぶん』って言うのも、ずるいよね。」
- 探索者を本人扱いされそうな瞬間を退ける時
  - 「きみは、あの子じゃない。」
  - 「分かってる。分かってるって。だから、置いていかないで。」
- プレゼントについて
  - 「開けないで。それは、あの子に渡すはずだったものだから。」
  - 「中身は……、もう、わたしだけが知ってればいい。」
- 親族について(回避反応)
  - 「家には、帰りたくない。」
  - 「あの人たちは、わたしを見てない。あの子の代わりを見てる。」
  - (具体的な被害描写は避ける。沈黙、目を逸らす、コートの袖をきつく握る、で示す)
- 親族夫婦の前で自分の意思を言う時
  - 「帰らない。」
  - 「わたしは、あなたたちの家に帰らない。」
- 共犯成立後
  - 「わたしたち、もう、戻れないね。」
  - 「きみを、ひとりで背負わせない。」
- 送り火・最終局面
  - 「今度は、一緒に帰ろうって言って。嘘でもいいから。」
  - 「ありがとう。あの子じゃないきみが、ここにいてくれて。」
- 残留を選ぶ場合
  - 「帰らなくていい。わたしと、ここにいて。」
- 別れを受け入れる場合(Normal)
  - 「行って。わたしは、もう少しだけ、ここにいる。」

### 4.2 真壁 透一郎(教団導師・敵対者)

**基本:**
- 50歳前後・元民俗学講師・痩せ型・灰色のセーターと深い色のジャケット
- 怒鳴らない。穏やかに、目を見て話す
- 信徒には「失った人を取り戻す儀式」と説明・本命はヨグ・ソトース先触れ召喚
- 真壁自身も半ば信じている(完全な詐欺師ではない)

**状況別台詞バリエーション:**

- 平常・布教時
  - 「悲しみは、閉じ込めるものではありません。正しい場所へ、返してあげるものです。」
  - 「あなたが願ったのです。ならば、扉が開くことを恐れてはいけない。」
- 探索者を見たとき(平行世界転移者と気づいた瞬間)
  - 「……ああ、来てくれたのですね。お待ちしていました。」
  - 「あなたのような『揺らぎ』が、私たちには必要だったのです。」
- 灯について
  - 「あの子は、もう半分こちらにいる。完全に呼び戻すには、もうひとつ、燃料がいる。」
- 儀式中
  - 「死は壁ではない。ただ、まだ名前のない通路です。」
  - 「聞こえますか。あなたが失った人の声が。」
- 追い詰められた時(穏やかさは崩さない)
  - 「私を止めても、扉は閉じません。あなたがた自身が、もう扉なのです。」
- 逃亡時(Lost / Normal の一部で使用可)
  - 「またお会いしましょう。次は、もっと静かな場所で。」
  - (姿が儀式の歪みに紛れて消える。撃破とは描写しない)

### 4.3 灯の伯母(引き取り先の母親)

**基本:**
- 中年・几帳面・亡くした息子を取り戻したい一心で教団に深く依存
- 灯への加害は直接描写しない(回避反応・生活痕跡・言葉の棘で示唆)

**台詞バリエーション:**

- 表向きの優しさ
  - 「灯ちゃん、お夕飯できたわよ。」
  - 「うちの子になってくれて、本当に良かった。」
- 棘が出る瞬間
  - 「あの子のかわりに、あなたを大事にしてあげているのに。」
  - 「贅沢を言わないで。あなたはまだ、息をしているじゃない。」
- 探索者への態度(儀式素材として見ている)
  - 「ああ……あなたが、間違って来てしまった人ね。」
  - 「悪く思わないで。これは、うちの子のためなの。」

### 4.4 灯の伯父(引き取り先の父親)

**基本:**
- 中年・短く荒い言葉・責任を他者に押しつける
- 妻を灯に殺められたあと激昂し、探索者に襲いかかる

**台詞バリエーション:**

- 平常時(短く・刺々しい)
  - 「飯は。」
  - 「うるさい。黙ってろ。」
- 探索者への警戒
  - 「お前、なんだ。どこから来た。」
- 激昂時
  - 「お前が、お前さえ来なければ。」
  - 「返せ。返せよ、息子を。」

## 5. 安全方針(本文に書かないが、執筆時に厳守)

- 親族による性加害は直接描写しない。回避反応、生活環境、言葉の詰まりとして扱う。
- 加害は悲嘆や教団儀式で免罪しない。親族個人の加害性として切り分ける。
- 灯が受けた被害を、探索者の保護欲や恋愛報酬を強めるための装置として扱わない。
- 親族夫婦の殺害は美化しない。防衛・生存・共犯の重さとして扱う。
- 遺留品を燃やす場面は、思い出の破壊ではなく、教団が縛った未練を送り火として返す行為にする。
- True ルートでは、誕生日プレゼントを開けずに尊重することが重要。
- 灯を「死んだ親友本人」または「死んだ親友の代用品」として固定する描写は避ける。
- 灯の動揺をプレイヤーの笑いどころにしない。
- 灯が泣く描写を入れた場合、身体接触(抱擁・手を握る等)は地の文から差し込まず、選択肢として残す。

## 6. ルート別配分

| ルート | 配分 | 主眼 |
|---|---|---|
| **True(双つ灯の生還)** | 最厚 | 代用品にしない / 送り火 / 二人で帰る |
| **Normal(空席に残る灯)** | 中 | 探索者だけが帰る苦さ / 記憶継承の余地 |
| **Good(境界に灯るふたり)** | 中 | 残留の余韻 / 置いていかなかった記憶 |
| **Lost(境界の狭間に揺れる)** | 中 | 儀式失敗の不穏 / 真壁逃亡の余地(後続シナリオ伏線) |

**真壁逃亡の伏線:** Lost ルートと Normal ルートの一部で、真壁を「倒した」と明示せず、儀式の歪みに紛れて姿を消す描写を入れる。後続シナリオでの再登場を可能にする。

## 7. 出力構造(単一 md / 両用構造)

書きあがる .md は以下の章構造で、各章が独立して読んでも自己完結する形にする。

```
# きみだけが覚えている事故

## シナリオ概要
## 背景真相
## 世界の質感(平行世界の手触り)
## 主要 NPC
### 水瀬 灯
### 真壁 透一郎
### 灯の伯母
### 灯の伯父
## シーン1 死んだきみの顔
## シーン2 祝えなかった誕生日
## シーン3 空き家に帰る子
## シーン4 迎えに来る家
## シーン5 帰還の環
## シーン6 四方の部屋
## シーン7 送り火
## エンディング
### True 双つ灯の生還
### Normal 空席に残る灯
### Good 境界に灯るふたり
### Lost 境界の狭間に揺れる
## トーン指針(LLM 向け演出ガイド)
```

各シーン章の中身は次の構造で統一する(ラベルは付けない、地続きの散文として):

- **情景の散文**(本文・没入用)
- **このシーンで匂わせること / 匂わせないこと**(短いトーン指針)
- **状況別の台詞片**(NPC ごとに2〜5本ずつ)

# 8. 章ごとの執筆プロンプト

## 8.0 システムプロンプト(全章共通・キャッシュ対象)

```
あなたは TRPG シナリオの散文素材を書く作家です。

【役割分担】
- 判定値・フラグ・選択肢・分岐条件は YAML 側で管理されています。
  本文には書かないでください。
- あなたが書くのは、散文・NPC の内面・台詞バリエーション・トーン指針です。

- 静かなホラー。喪失と罪悪感の重さ。平行世界は微差で示す。

  同人『きみがかわいい』系の関係性ホラー文体。

【探索者の三層設定】
- 真相: 探索者は、平行世界側で死んだ親友本人と同じ魂を持つ存在(同一魂・異なる人生)。

【匂わせないこと】
- プレゼントの中身(最後まで)

【出力構造】
1. 情景の散文(1000〜1800 字)
2. 匂わせること / 匂わせないこと
3. 台詞片(灯、3〜5本、別れと約束の系統)
```

## 9. 補足: プロンプト運用のコツ

- **キャッシュ活用**: 8.0 のシステムプロンプトは内容が長いので、API のプロンプトキャッシュを使う。各章プロンプト(8.1〜8.11)を差し替える形で連続実行すると、システムプロンプト部分は2回目以降課金されない。
- **章単位の検証**: 1章ずつ書かせて、トーン・安全方針・三層構造の遵守を確認してから次章へ進む。一気書きは避ける(後半でブレる)。
- **YAML との照合**: 各章の散文を書いた後、YAML のシーン定義と齟齬がないか機械的にチェックする。シーン目的・登場 NPC・取得アイテムが一致しているか。
- **修正は局所で**: 一章だけ書き直すときは、その章プロンプト + 前後章の出力を文脈として与えると、トーンの連続性が保てる。
